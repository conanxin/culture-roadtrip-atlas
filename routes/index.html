数据规范

v1.4.7 全面通用化 · 独立 docs/ROUTE_DATA_SPEC.md / 独立 docs/ROUTE_PAGE_TEMPLATE.md / routes-manifest.json

当前路线数据统一使用 CSV / GeoJSON / GPX 三格式。每个点位必须包含 18 个字段：

  • segment_id · 段落编号（S01–S10 或 S01–S09）
  • source_level · 来源等级（A_official_oedw / B_partner_or_education / C_chinese_media / D_reconstructed_for_travel / E_project_internal）
  • coordinate_precision · 坐标精度（city / landmark / region / approximate / unknown）
  • replication_feasibility · 复刻可行性（high / medium / low / not_recommended / reading_only）
  • difficulty · 旅行难度（easy / medium / hard / extreme / mixed / n/a）
  • risk_note · 风险提示

所有格式 metadata / desc 中必须明确标注：非原始 GPS 轨迹 + 非导航。所有路线共享 routes-manifest.json 集中登记。

详细规范：docs/ROUTE_DATA_SPEC.md · 页面模板：docs/ROUTE_PAGE_TEMPLATE.md · 数据目录说明：data/routes/README.md

为什么需要多路线规范？

  • 一致性：所有路线使用同一套字段、同一套 metadata 声明、同一种免责声明。
  • 可维护性：脚本与查看器只写一份，新增路线无需重写。
  • 可审校性：任何路线都遵守“非导航 / 非原始轨迹” 边界，不冒充精确数据。
  • 可扩展性：未来山西古建、北京周边等路线可直接复用。

关于路线数据

路线数据是行旅图谱项目中"可审校、可复用、可沉淀"的部分。我们相信：

  • 数据优于攻略：表格化、可分析的数据比一次性攻略更有价值。
  • 粗点优于精确：承认"不知道"比伪造精确坐标更诚实。
  • 非导航优于误用：明确标注"非导航"比让数据被误用更负责。
  • 纯静态优于依赖：CSV / GeoJSON / GPX 都是纯文本格式，不依赖任何专有软件。
  • 文化复刻优于原始 GPS：我们整理的是文化意义，不是 GPS 精度。

详细字段说明、数据限制、使用方法见 data/routes/README.md。
