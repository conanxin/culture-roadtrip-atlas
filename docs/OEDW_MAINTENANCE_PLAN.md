## 2. 维护期原则

1. **不再以继续扩写为目标**。后续改进方向：验证 / 维护 / 补图 / 修正。
2. **所有变更写入 CHANGELOG**。每次 commit 必须说明改了什么、为什么改。
3. **不稳定信息必须标注来源**。不得把推测写成确定事实。
4. **实地验证优先于文案完善**。开放状态、交通等信息必须先有来源才能写进正文。
5. **版本号反映改动性质**：
   - `v1.5.x`：维护期 patch（错字 / 断链 / SEO 微调 / 脚本体检增强）
   - `v1.6.0`：首次加入自有实拍图或授权图像
   - `v1.7.0`：首次完成一条真实实走验证线
   - `v1.8.0`：完成多段实走反馈与交通核验
   - `v2.0.0`：全线或核心段落完成实地验证后

## 3. patch 规则

### 允许直接进入 v1.5.x 的改动

- 错字修正
- 断链修复
- SEO 描述微调
- 表格样式微调（移动端可读性）
- 移动端可读性修复
- 参考资料链接更新（断链换新链接）
- 已知事实口径修正（有可靠来源）
- 图片授权状态更新
- 实拍图元数据补充
- 脚本体检规则增强

### 需要讨论后才能改的

- 新增路线段落
- 修改起终点或里程
- 修改 Milestone 编号对应关系
- 修改风险等级评估
- 修改各段交通信息

### 禁止未经实地/可靠来源验证直接改的

- 点位开放状态
- 真实交通可达性（公交/包车/徒步）
- 具体住宿建议
- 乡村 / 山地 / 边境路线可走性
- 每日耗时（步行距离/时间）
- 推荐自驾 / 徒步难度评级
- 是否适合普通旅行者逐点复刻

## 4. 实地验证边界

以下信息必须有以下来源之一，才能写进正文或替换 placeholder：

| 信息类型 | 接受来源 |
|---------|---------|
| 点位开放状态 | 官方公告 / 景区官网 / 电话确认 |
| 交通可达性 | 公交/铁路官网 / 地图 API 实时查询 / 近期实走记录 |
| 住宿建议 | 实地住宿经历 / 可靠旅行者近期反馈 |
| 徒步难度 | 当地向导或有过实地经验者的描述 |
| 实拍图 | 自有拍摄 / 明确书面授权 / 公开 CC 许可 |
| 每日分段耗时 | GPS 轨迹记录 / 实走日志 / 多次平均 |

|---------|---------|---------|

| 图片授权链接 | 每次新增图片时 | 入库前检查 |

记录字段：`checked_at` · `status` · `method` · `result` · `note`

## 6. 图片授权规则

### 当前原则

- ✅ 允许：自有实拍 · 明确书面授权 · 公开可复用许可（CC BY / CC BY-SA）
- ❌ 禁止：复制 National Geographic / OEDW 官网图片 · 下载第三方图片 · 来源不明图片

### 入库必须记录

- `filename`：文件名
- `source`：图片来源
- `photographer / owner`：摄影者或权利人
- `license`：许可类型
- `permission_status`：授权状态（已授权 / 待确认 / 不可用）
- `related_milestone`：关联 Milestone（如有）
- `related_segment`：关联路线段（如有）
- `added_at`：入库日期
- `note`：备注

### 图片状态

- 待实拍
- 已实拍待筛选
- 已授权
- 需署名
- 不可用
- 待替换
