# 行旅图谱 · 路线页面模板 v0.9

**适用版本**: v0.9+
**最后更新**: 2026-07-04

本模板定义未来每条路线页面的标准结构。所有新路线（无论是"已上线完整路线"还是"规划中骨架页"）都应遵循本模板的章节结构，以保证路线库的一致性。

---

## 1. 标准页面结构

每条路线页面应包含以下章节（按顺序）：

### 1.1 顶部 Hero（必填）
- 路线名称（如：「山西古建自驾线」）
- 副标题（一句话路线定位）
- 状态标签（已上线 / 规划中 / 待实地复盘 / 已实地复盘）
- 关键标签（核心主题词）

### 1.2 路线总览（必填）
- 推荐天数
- 路线类型
- 总里程（估算）
- 起终点城市列表
- 核心主题列表
- 路线阶段说明

### 1.3 路线示意图（推荐，完整路线必填）
- 纯 SVG 路线图
- 节点：起终点 + 关键城市
- 闭环或开放式
- 可选：Day 高亮按钮、主题路径切换

### 1.4 每日行程（完整路线必填）
- Day 1 - Day N 时间线
- 每天含：路线 / 车程 / 住宿 / 主题 / 必看景点 / 美食
- 完整路线：详细日程 + 替代方案
- 规划中：仅路线骨架

### 1.5 城市章节（推荐）
- 城市定位
- 路线角色
- 代表景点
- 代表美食
- 关键词
- 每个城市 1-2 段历史背景

### 1.6 景点卡片（完整路线必填）
- 景点名称
- 时代
- 类型
- 推荐等级
- 一句话价值
- 为什么去
- 现场怎么看
- 重点细节
- 实用信息（开放时间/门票/停车）
- 导游词（800 字左右，重点景点）
- 来源字段（v0.6+）

### 1.7 随行导游词（完整路线必填）
- 每个重点景点独立段落
- 包含：建筑背景 + 历史意义 + 观看方法 + 现场感受
- 摘句版（v0.7+）

### 1.8 美食模块（必填）
- 每个城市 2-5 个代表美食
- 简短描述

### 1.9 行前书影音（必填）
- 行前 7 天学习计划（v0.3+）
- 三种预习方案：30 分钟速览 / 2 小时标准 / 7 天深度（v0.4+）
- 推荐资料列表

### 1.10 知识图谱（推荐）
- 时间轴（如辽代时间轴）
- 五京体系（如辽五京）
- 形制比较（如辽塔形制）
- 核心概念词条

### 1.11 资料来源索引（v0.6+ 必填）
- 官方资料 / 博物馆资料 / 纪录片 / 书籍 / 研究 / 旅行整理
- 每条来源含：名称 / 类型 / 关联景点 / 用途 / 可信度

### 1.12 事实核对标签（v0.6+ 必填）
- 出发前需核对
- 旅行整理
- 资料整理

### 1.13 版本记录（必填）
- 页面版本
- 最近更新日期
- 资料状态

## 2. 字段填写规范

### 必填字段（无则页面不完整）
- `name`：路线名称
- `shortName`：短名（用于卡片）
- `oneLine`：一句话说明
- `status`：已上线 / 规划中 / 待实地复盘 / 已实地复盘
- `days`：天数
- `route`：起终点
- `themes`：核心主题
- `tags`：标签
- `url`：页面链接

### 推荐字段（增强体验）
- `suitableFor`：适合人群
- `progress`：当前进度（0-100）
- `categories`：分类（用于筛选）
- `isFeatured`：是否 Featured
- `isLive`：是否可点击进入

### 可选字段（后续补充）
- `totalDistance`：总里程
- `bestSeason`：最佳季节
- `createdDate`：创建日期
- `lastUpdated`：最后更新日期
- `realReviewer`：实地复盘人

## 3. 状态说明

### 已上线（status = "已上线"）
- 页面内容完整
- 景点信息经过基本核对
- 有完整导游词、来源、事实标签
- 按钮显示「进入行程」
- `progress >= 90%`

### 规划中（status = "规划中"）
- 仅有页面骨架
- 景点列表仅为候选
- 资料来源未核实
- 按钮显示「查看规划」
- `progress < 30%`

### 待实地复盘（status = "待实地复盘"）
- 路线已规划完成
- 等待实际走访
- 按钮显示「即将上线」
- `progress 30-80%`

### 已实地复盘（status = "已实地复盘"）
- 已根据真实旅行更新
- 是最可靠的路线状态
- 按钮显示「进入行程」
- `progress = 100%`

## 4. 视觉规范

### 风格
- 墨绿 / 米白 / 灰绿 / 暗金 色调
- 长卷式 / 文物图录感 / 克制 / 古朴
- 不要做成商业旅游网站风格

### 必备样式类
- `.trip-status`：路线状态标签
- `.trip-skeleton`：骨架页标识
- `.trip-template-section`：模板章节容器
- `.candidate-spot`：候选景点（规划中）
- `.planning-note`：规划中提示

### 响应式
- 移动端优先
- 卡片布局不溢出
- 长导游词可滚动

## 5. 新增路线的最小步骤

### 5.1 准备工作
1. 确定路线主题和方向
2. 准备至少 5 个候选景点
3. 收集路线起终点和大致节奏

### 5.2 创建文件
1. 创建目录 `trips/<route-id>/`
2. 创建 `trips/<route-id>/index.html`（可参考辽塔页）
3. 创建数据文件（如需要）

### 5.3 更新首页数据
1. 编辑 `assets/js/trips-data.js`
2. 添加路线条目
3. 设置 `status: "规划中"`
4. 设置 `progress: <small>`

### 5.4 更新 README 和 CONTENT_NOTES
1. 在 README 增加路线说明
2. 在 CONTENT_NOTES 增加开发进度

### 5.5 部署验证
1. commit + push
2. 等待 GitHub Pages 部署
3. 验证 HTTP 200
4. 检查关键词
